Install TonleDB

TonleDB can be installed on various Linux distributions. Choose your distribution below for installation instructions.

Ubuntu/Debian

Option 1: Download and install package directly

curl -L -O https://github.com/your-username/tonledb/releases/latest/download/tonledb_0.1.0_amd64.deb
sudo dpkg -i tonledb_0.1.0_amd64.deb
sudo apt-get install -f

Option 2: Add repository (recommended for automatic updates)

echo "deb [trusted=yes] https://repo.tonledb.org/apt/ /" | sudo tee /etc/apt/sources.list.d/tonledb.list
sudo apt update
sudo apt install tonledb

CentOS/RHEL/Fedora

Option 1: Download and install package directly

curl -L -O https://github.com/your-username/tonledb/releases/latest/download/tonledb-0.1.0-1.x86_64.rpm
sudo rpm -i tonledb-0.1.0-1.x86_64.rpm

Option 2: Add repository (recommended for automatic updates)

# For CentOS/RHEL
sudo yum-config-manager --add-repo https://repo.tonledb.org/yum/tonledb.repo
sudo yum install tonledb

# For Fedora
sudo dnf config-manager --add-repo https://repo.tonledb.org/yum/tonledb.repo
sudo dnf install tonledb

Other Linux Distributions

Using the generic installation script

curl -sSL https://repo.tonledb.org/install.sh | sudo bash

Post-Installation

Start the service

sudo systemctl start tonledb

Enable auto-start on boot

sudo systemctl enable tonledb

Check service status

sudo systemctl status tonledb

View logs

sudo journalctl -u tonledb -f

Note: Replace your-username in the URLs above with your actual GitHub username.

Configuration

The default configuration file is located at /etc/tonledb/tonledb.toml.

Usage

Once the service is running, you can interact with TonleDB via HTTP:

Health check

curl http://localhost:8080/health

SQL query

curl -X POST http://localhost:8080/sql \
  -H "Content-Type: application/json" \
  -d '{"sql": "SELECT * FROM users"}'

Key-Value operations

curl http://localhost:8080/kv/mykey
curl -X POST http://localhost:8080/kv/mykey -d "myvalue"
curl -X DELETE http://localhost:8080/kv/mykey

Document operations

curl -X POST http://localhost:8080/doc/users \
  -H "Content-Type: application/json" \
  -d '{"name": "John", "age": 30}'
